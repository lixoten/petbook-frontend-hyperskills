Description

Build the HTML forms for the Personal Data and Settings page, including different types of input: text, phone, email, checkboxes, and more!

Please use the settings.html file. As you can see on the designs, only the main content has been changed, so the navbar and footer will carry on. Now, the Hero section is the settings-section part and the aside element is the personal-data part of the settings page.

In profile.html and friends.html, update the navbar anchor links: settings-link should now direct to new settings.html.

Similarly, in settings.html for the navbar anchor links: settings-link should be directed to the current HTML file, the pet profile-link should lead to profile.html and friends-link should direct to friends.html.

Pay attention to the CSS rules and the elements in the Figma designs. The element names will be the ids of your HTML elements. Also, notice the element structure in Figma, which hints at your HTML element structure.

The personal-data and the settings-section inputs should be wrapped in form tags.

From the personal data aside on the settings page, the tests will only check the followings: Animal select, name input, about text area, birthday, image upload, and save data button. Also, from the settings section on the settings page, the tests will check all elements except the third parties checkbox.

Please, take a look at the designs made in Figma. This time you need to build only the design from Stage 4: Settings page.
